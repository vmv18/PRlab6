# Практична робота №6
## Налагодження React застосунків

### Відповіді на контрольні запитання:
1. Перевага React DevTools над звичайним інспектуванням DOM полягає в тому, що він показує ієрархію React-компонентів, їхні props, state та hooks, тоді як вкладка Elements відображає лише кінцевий HTML-результат.
2. Для відлову неоптимального рендерингу списків через неправильні ключі потрібно увімкнути "Highlight updates when components render" у налаштуваннях React DevTools, а також слідкувати за попередженнями типу "Encountered two children with the same key" у standard консолі браузера.
3. Алгоритм роботи з Breakpoint: відкрити вкладку Sources, знайти потрібний файл, клікнути зліва від номера рядка для встановлення блакитної позначки. Коли програма дійде до цього рядка, її виконання зупиниться, і можна буде дослідити поточні значення змінних у пам'яті.
4. Резервоване ключове слово для автоматичної зупинки виконання коду при відкритій панелі розробника - `debugger;`.
5. Дії в дебагері: Step Over (F10) переходить до наступного рядка коду без заходу всередину викликаних функцій, Step Into (F11) заходить всередину функції для її покрокового аналізу, а Step Out дозволяє швидко завершити виконання поточної функції та повернутися до місця її виклику.
